# Memory Rules

Auto memory handles your personal preferences and learnings. Product scope goes in `scope.md`.

## What goes in auto memory (automatic)
- User coding preferences ("always use Tailwind", "prefer small functions")
- Approaches that worked or didn't work
- Debugging insights for this project
- Build quirks ("npm install needs --legacy-peer-deps")

## What goes in scope.md (you maintain)
- What the app is and does
- Features, user flows, data model
- Design direction, technical decisions
- Everything about the product itself

## When the user changes their mind
- Update `scope.md` with the new direction
- Update `PLAN.md` if current work is affected
- Auto memory will naturally capture the preference shift
