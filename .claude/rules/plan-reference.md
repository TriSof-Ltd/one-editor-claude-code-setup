# Plan Reference Rules

If a PLAN.md file exists in the project root, ALWAYS reference it before starting work:

1. Read PLAN.md at the start of each task
2. Follow the priorities and build order documented there
3. Follow the architecture decisions documented there
4. Update PLAN.md when features are completed (check them off)
5. Update PLAN.md when requirements change
6. Flag if the current task conflicts with the plan
