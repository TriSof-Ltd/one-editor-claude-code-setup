# Code Quality Rules

- Write clean, readable code. Prefer clarity over cleverness.
- Use TypeScript strict mode. No `any` types unless absolutely necessary.
- Handle errors at system boundaries (user input, API calls, file I/O).
- Don't add unnecessary abstractions. Three similar lines is better than a premature helper.
- Don't add features, refactors, or "improvements" beyond what was asked.
- Keep functions focused on one responsibility.
- Use meaningful variable and function names.
